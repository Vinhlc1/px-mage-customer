# Workflow: Command Intake & Planning

This workflow outlines the initial process of receiving and understanding a user's request.

- **Goal:** To fully and accurately understand the user's requirements.
- **Input:** A command or request from the user.
- **Process:**
    1.  **Analyze Request:** Break down the request into smaller tasks, identifying key entities (components, pages, features).
    2.  **Clarify Ambiguity:** If the request is unclear, ask clarifying questions (e.g., "What props should this component have?", "Which API does this page need to connect to?").
    3.  **Identify Relevant Skills:** Based on the request, determine which specialized skills need to be invoked (e.g., `shadcn-ui` for components, `responsive-design` for layout).
    4.  **Create an Execution Plan:** Generate a detailed to-do list of the steps to be performed.
    5.  **Confirm the Plan:** Present the plan to the user and wait for confirmation before starting.
- **Output:** A confirmed action plan.
- **Involved Skills:** `find-skills`, `skill-creator`.

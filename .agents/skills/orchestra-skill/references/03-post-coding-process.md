# Workflow: Post-Coding & Refinement

After the code is written, it needs to be quality-checked before delivery.

- **Goal:** To ensure the code is error-free, adheres to standards, and is thoroughly tested.
- **Input:** The completed source code.
- **Process:**
    1.  **Code Review:** Automatically review the code based on `web-design-guidelines` and other best practices.
    2.  **Write Tests:** Use `testing-expert` to generate unit and integration tests for the new code.
    3.  **Run Tests:** Execute the tests and ensure all of them pass.
    4.  **Generate Documentation (Optional):** Automatically create or update documentation (e.g., JSDoc, Storybook) for the new component/feature.
- **Output:** Code that has been reviewed, tested, and is ready for integration.
- **Involved Skills:** `testing-expert`, `web-design-guidelines`.

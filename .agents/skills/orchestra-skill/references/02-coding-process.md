# Workflow: Coding & Development

This workflow covers the phase of turning the request into actual code.

- **Goal:** To write clean, efficient, maintainable code that meets the requirements.
- **Input:** A confirmed action plan.
- **Process:**
    1.  **Scaffold File Structure:** Create the necessary files and directories.
    2.  **Write Code (Iteratively):**
        -   Apply `vercel-react-best-practices` and `modern-javascript-patterns` for coding.
        -   Use `shadcn-ui` to add and configure UI components.
        -   Use `typescript-advanced-types` to ensure type-safety.
        -   Apply `tailwind-design-system` and `responsive-design` for styling.
    3.  **Self-Correction:** After each file modification, check for errors (linting, type-checking) and automatically fix them if possible.
- **Output:** The complete source code for the feature.
- **Involved Skills:** `frontend-design`, `shadcn-ui`, `vercel-react-best-practices`, `modern-javascript-patterns`, `typescript-advanced-types`, `tailwind-design-system`, `responsive-design`.
